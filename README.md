# Waterpark　飲料水のECサイト

## 概要
職業訓練校の卒業制作として、4名チームで開発したECサイトです。<br>
Java（Spring Boot）を用いた堅牢なシステム構築を軸に、Pythonを活用したデータ分析・レコメンデーション機能を統合し、ユーザーに最適化された体験を提供することを目指しました。

---

## 担当範囲（実装機能）
- **認証系:** ログイン・会員登録・会員情報変更・ログアウト（セッション管理とセキュリティの実装）
- **退会処理:** ユーザーデータの整合性を保った退会ロジックの実装
- **Python統合:** 推薦アルゴリズムを用いた「おすすめランキング」「新着・関連商品」の抽出・表示機能

---

## 使用技術
- **バックエンド:** Java 8.x / Spring Boot
- **データ処理:** Python（データ分析・レコメンドロジック）
- **データベース:** MySQL
- **その他:** Git/GitHub

---

## 注力したポイント

### 1. データ駆動型のアルゴリズム設計（Python）
- **スコアリングの工夫:** 単純な新着順ではなく、「コスパ（容量/価格）」「新着度（日数経過による減衰）」「希少性（在庫僅少による加点）」を独自に重み付けして算出するアルゴリズムを実装しました。
- **スケーラビリティへの配慮:** Pythonで計算処理を行うことで、SQL単体では複雑になりがちな集計ロジックを切り分け、メンテナンス性を高めました。

### 2. バックエンドとフロントの疎結合化（API実装）
- **JSONベースのAPI設計:** API経由でのデータ取得を行うことで、フロントエンドとバックエンドの役割を分離し、データ更新時の柔軟性を確保しました。

### 3. Spring Bootによる堅牢な設計
- Javaを用いた大規模開発を見据え、MVCモデルを意識したディレクトリ構成とセキュアな認証処理を実装しました。

---

## 苦労した点・解決プロセス
- **PythonとJavaの連携:** 異なる言語間でのデータ受け渡しにおいて、処理速度と整合性の維持に工夫を凝らしました。DBを介したデータの受け渡し設計を徹底し、Python側の算出結果を安定して画面へ反映させる仕組みを構築しました。
- **会員情報の整合性管理:** 会員登録・変更・退会の一連の流れにおいて、エラーハンドリングを丁寧に行うことで、システム全体の安定性を高めました。
